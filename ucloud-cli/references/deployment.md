# 部署和创建默认值

- 当用户只表达部署目标，如“部署我的 WEB 服务”“把这个站点上线”或“在 UCloud 上把这个应用上线”，先结合产品文档、当前上下文和用户需求明确部署形态，再选择具体云资源。
- 对于 WEB 服务或通用应用部署请求，当用户未明确选择基础设施形态时，从符合需求的最小资源集开始，不预先假设某个主机产品。
- 如果后续需求超出初始方案，先说明切换原因，再迁移到满足需求的最小更广资源集。
- 除非用户明确要求某种计费方式，否则默认优先按小时预付费。
- 如果目标服务或配置不支持按小时预付费，尝试按小时后付费，然后是按月预付费。
- 如果按小时预付费、按小时后付费和按月预付费均不可用，停止并询问用户是否尝试按年预付费。
- 资源创建成功后，摘要展示关键访问和清单字段，如登录用户名、登录凭据、内网 IP、外网 IP、资源 ID、Region、Zone、规格、计费模式以及已绑定的安全组或网络信息。
- 为资源生成初始密码时，仅使用大写字母、小写字母和数字。除非用户明确要求，否则不使用特殊字符。
